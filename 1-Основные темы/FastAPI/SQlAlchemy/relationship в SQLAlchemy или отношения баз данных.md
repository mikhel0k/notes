
Две таблицы связываются на уровне базы данных при помощи ForeignKey, причем зачастую односторонне, то-есть одна таблица знает о другой, а вторая вот вообще не в курсе. Например:
```
  book_id: Mapped[int] = mapped_column(ForeignKey('books.id'), nullable=False)
```

В SQLAlchemy для этого существуют relationship поля, только тут это реализуется так что обе таблицы знают друг о друге и через обращение к этому полю можно вытаскивать информацию из другой таблицы. Реализуется это через JOIN запросы, но неявно. Например в классе Book я задаю следующее поле:
```
physical_books: Mapped[list["PhysicalBook"]] = relationship("PhysicalBook", back_populates="book")
```
Теперь у меня класс Book знает что у него есть связь с PhysicalBook, но вот PhysicalBook пока что не в курсе что у него есть связь, чтобы он узнал об этом необходимо прописать:
```
book: Mapped["Book"] = relationship("Book", back_populates="physical_books")
```
Теперь оба класса знают друг о друге, в конструкцию relationship необходимо передавать два параметра, название класса с которым делается связь и поле через которое в другом классе реализована как раз эта самая связь. По умолчанию создается связь много-к-одному, если необходимо создать связь один-к-одному то нужно ставить флаг `uselist=False`.
